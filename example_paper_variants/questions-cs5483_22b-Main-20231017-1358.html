Both DBSCAN and OPTICS are [[1]] clustering methods. Similar to DBSCAN, OPTICS requires [[2]] input parameters \(\epsilon\) and \(\operatorname{MinPts}\) as DBSCAN does. Unlike DBSCAN, OPTICS can find clusters of [[3]] densities because of the following modifications:

  • Instead of simply checking whether a data point is a [[4]], OPTICS calculates the [[5]] in addition to reflect the density of the region containing the data point.
  • Instead of growing the cluster to any [[6]], it visits points with [[7]] [[8]] first.

More precisely, given a data point \(\boldsymbol{p}\) is a [[4]], it calculates [[9]] \(\epsilon'\leq \epsilon\) such that the \(\epsilon'\)-neighborhood contains [[10]] than or equal to \(\operatorname{MinPts}\) data points. A priority queue is used to store its [[6]] \(\boldsymbol{q}\) according to the [[8]], which is [[15]] between the [[5]] of \(\boldsymbol{p}\) and the [[11]] from \(\boldsymbol{p}\) to \(\boldsymbol{q}\).
